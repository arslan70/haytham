## RULES

1. **NO IMPLEMENTATION CODE** — Stories are specifications, not implementations
2. **GHERKIN FORMAT** — Use Gherkin scenarios for acceptance criteria
3. **THREE-CASE COVERAGE** — For each capability, cover: happy path, error case, access control
4. **INPUT VALIDATION** — Specify validation rules for all user inputs (types, constraints, required fields)
5. **PERMISSION TRACEABILITY** — For every write operation, list the required permission and cross-reference the data model's permission matrix
6. **ADAPT TO STACK** — Use the stack from architecture decisions. Do NOT assume REST vs GraphQL vs CLI
